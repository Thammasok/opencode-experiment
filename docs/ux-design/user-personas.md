# User Personas — MicroBlog Platform

---

## Persona 1 — The Casual Reader

**Name:** Maya, 27 — Marketing Coordinator
**Goal:** Stay informed about topics she cares about without creating content
**Behaviour:** Browses explore feed and specific profiles; rarely logs in; consumes more than she produces

**Pain points:**
- Doesn't want to register just to read public content
- Dislikes friction-heavy onboarding before seeing value
- Gets overwhelmed by dense, cluttered feeds

**Needs from MicroBlog:**
- Public explore feed accessible without account
- Clean, readable post cards with clear authorship
- Easy path from visitor → member when she finds value

**Device:** 80% mobile, 20% desktop

---

## Persona 2 — The Active Poster

**Name:** Kai, 32 — Software Engineer
**Goal:** Share technical opinions, build an audience, engage in conversations
**Behaviour:** Posts daily; replies to threads; checks notifications frequently; tracks like/follower count

**Pain points:**
- Wants fast compose — as few taps as possible to post
- Needs instant feedback that a post published successfully
- Frustrated when compose area loses draft on navigation

**Needs from MicroBlog:**
- Persistent compose shortcut accessible from anywhere
- Character counter always visible while typing
- Notification badge that updates in real time

**Device:** 50% mobile, 50% desktop

---

## Persona 3 — The Social Connector

**Name:** Priya, 24 — Journalism Student
**Goal:** Discover new voices, build a curated following list, participate in discussions
**Behaviour:** Heavy use of explore and search; follows/unfollows frequently; reads threads fully

**Pain points:**
- Wants to understand who someone is before following (needs profile preview)
- Loses track of who she already follows when browsing
- Wants to see conversation threads in context, not isolated posts

**Needs from MicroBlog:**
- Clear follow/unfollow toggle state on every profile and post card
- Explore feed with search across posts and usernames
- Thread view that shows full parent-reply chain

**Device:** 65% mobile, 35% desktop
